Chapter 4 : SetUp An AEM Instance

Before initiating the instance creation process, let's delve into the distinct roles of two unique instances—author and publish—which are essential for end-to-end development.

December 10 ,2023

  • Author Instance :

    1. The author instance is exclusively dedicated to content and asset authoring, a responsibility restricted to administrators and content authors.
    2. Typically, it is hosted on port 4502.
    3. Creating an AEM instance involves naming the AEM JAR file as cq65-author-p4502.jar.
    4. Content becomes accessible to the public or end users only after activation or publication.
    5. The button below facilitates the publishing of content to the publish environment.

    • AEM Publish Instance:

    1. The publish instance primarily functions as an end-user website, dedicated to the display of content without content editing or authoring capabilities.
    2. Typically, it is hosted on port 4503.
    3. To set up an AEM instance, the AEM JAR file must be named cq65-publish-p4503.jar.
    4. Content becomes accessible in the publish instance immediately upon activation, publication, or replication from the author instance.
    5. Unpublishing content is also an option, allowing the removal of content from the live site or restricting its availability to end users.
    6. The publish instance is commonly referred to as the publisher.
    7. In scenarios anticipating a large number of users, multiple publishers can be employed for a single author instance to distribute the page access load.
